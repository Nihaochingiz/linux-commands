# Основные команды Linux

## Работа с директориями

| Команда | Описание |
|---------|---------|
| `pwd` | Показать текущую рабочую директорию |
| `ls` | Вывести список файлов и директорий в текущей рабочей директории |
|
| `ls -l` | Показать подробный список с правами доступа, владельцем, размером и датой изменения |
| `ls -a` | Вывести все файлы, включая скрытые (начинающиеся с точки) |
| `ls /opt` | Показать содержимое директории /opt |
| `ls ..` | Показать содержимое родительской директории |
| `cd /home` | Перейти в директорию /home |
| `cd ../..` | Перейти на две директории вверх |

## Просмотр содержимого файлов

| Команда | Описание |
|---------|---------|
| `cat .bash_logout` | Прочитать содержимое файла .bash_logout |
| `head` | Посмотреть начало файла (по умолчанию 10 строк) |
| `tail` | Посмотреть конец файла (по умолчанию 10 строк) |
| `tail -n 2 .bashrc` | Посмотреть последние 2 строки файла .bashrc |
| `tail -f path/to/file` | Вывести последние строки файла и ожидать появления новых (полезно для просмотра логов) |
| `less` | Пейджер для просмотра содержимого файла в режиме чтения (похож на текстовый редактор) |

## Поиск в файлах

| Команда | Описание |
|---------|---------|
| `grep aliases .bashrc` | Поиск текста "aliases" в файле .bashrc |
| `grep pattern file` | Поиск определенного текста по файлу или файлам |

## Редактирование файлов (vim)

| Действие | Команда |
|---------|---------|
| Открыть файл | `vim solution` |
| Перейти в режим вставки | `Esc` затем `i` |
| Ввести текст | `Hello, World!` |
| Выйти из режима вставки | `Ctrl+[` |
| Сохранить и выйти | `:wq` |

## Перенаправление потоков

| Команда | Описание |
|---------|---------|
| `cat source > /tmp/result` | Прочитать содержимое файла source и перенаправить его в файл /tmp/result (создает файл, если его не существует, или перезаписывает существующий) |

## Работа с файлами

| Команда | Описание |
|---------|---------|
| `touch empty-file` | Создать файл |
| `rm empty-file` | Удалить файл |
| `touch file` `mv file renamed-file` | Создать файл и переименовать его |
| `cp renamed-file renamed-file-copy` | Копирование файла |
| `cat languages \| sort \| head -n 2` | Отсортировать содержимое файла в алфавитном порядке и вывести первые две строки |

## Работа с директориями (создание и удаление)

| Команда | Описание |
|---------|---------|
| `mkdir my-dir` | Создать директорию |
| `mkdir -p one/two/three` | Создание директорий рекурсивно (создает все промежуточные) |
| `rm -r my-dir` | Удаление директории (рекурсивно) |
| `rm -rf one` | Удаление всей директории без запросов подтверждения |

## Переменные окружения

| Команда | Описание |
|---------|---------|
| `env` | Посмотреть установленные переменные окружения |
| `echo $HOME` | Вывести значение переменной HOME |
| `export HOME=/tmp` | Установить новое значение переменной HOME |
| `export MAIL=/var/tmp NEW_VAR=value` | Установить несколько переменных одновременно |
| `cd` `pwd` | После изменения HOME перейти в домашнюю директорию и проверить текущий путь |

## Поиск в истории команд

| Команда | Описание |
|---------|---------|
| `ctrl + r` | Открыть обратный поиск по истории команд |
| (после ctrl+r) `mail` | Найти команду, содержащую слово "mail" |

## Информация о пользователе и системе

| Команда | Описание |
|---------|---------|
| `whoami` | Выяснить имя текущего пользователя |
| `id` | Посмотреть идентификатор пользователя, группы и другую информацию |
| `id nobody -u` | Вывести только UID пользователя nobody |
| `ps` | Отчет о работающих процессах текущего пользователя |
| `ps aux` | Информация о всех процессах, какой процесс и под каким пользователем запущен |

## Повышение привилегий (sudo)

| Команда | Описание |
|---------|---------|
| `sudo touch /etc/myfile` | Выполнить команду с правами суперпользователя |
| `sudo -u nobody mkdir /tmp/test` | Выполнить команду от имени пользователя nobody |
| `echo 'Hello, World!' \| sudo tee config/myfile` | Записать данные в файл с правами суперпользователя |

**Примечание:** Крайне не рекомендуется использовать root на регулярной основе и ни в коем случае не входить под ним в систему. Суперпользователь — это прямой доступ ко всему и большая дыра в безопасности системы.

## Права доступа к файлам

Права доступа:
- `r` — чтение (read)
- `w` — запись (write)
- `x` — исполнение (execute)

| Команда | Описание |
|---------|---------|
| `chmod 511 myfile` | Установить права: для владельца — чтение и исполнение; для группы и остальных — только исполнение |
| `chmod u+x program` | Добавить право на исполнение для владельца файла |
| `./program` | Запустить исполняемый файл |
